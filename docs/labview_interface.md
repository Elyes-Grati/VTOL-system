# LabVIEW Supervisory Interface

**File:** `labview/FinalFinal.vi`  
**LabVIEW version:** 18.0 (required for opening the VI)

## Overview

The LabVIEW VI is the supervisory controller and human-machine interface for
the VTOL rig. It connects to the Arduino over a VISA serial port, receives the
state vector, executes the LQR-I control law, and sends ESC commands back to
the Arduino at 100 Hz.

## VI Architecture

### Front Panel Controls

| Control | Type | Description |
|---------|------|-------------|
| VISA Resource | String | COM port selection (e.g., COM3, /dev/ttyACM0) |
| Setpoint e | Numeric | Target elevation angle (rad) |
| Setpoint Ψ | Numeric | Target yaw angle (rad) |
| Enable Control | Boolean | Arms the control loop |
| Stop | Boolean | Graceful shutdown |

### Front Panel Indicators

| Indicator | Type | Description |
|-----------|------|-------------|
| e chart | Waveform | Real-time elevation angle |
| θ chart | Waveform | Real-time roll angle |
| Ψ chart | Waveform | Real-time yaw angle |
| ė chart | Waveform | Elevation rate |
| θ̇ chart | Waveform | Roll rate |
| Ψ̇ chart | Waveform | Yaw rate |
| ESC1 µs | Numeric | Current ESC1 command |
| ESC2 µs | Numeric | Current ESC2 command |

### Block Diagram Signal Flow

```
Serial RX (VISA Read)
        │
        ▼
  String Parse (CSV split)
        │
        ▼
  [e, θ, Ψ, ė, θ̇, Ψ̇]  (1D array, 6 elements)
        │
        ▼
  Integral Accumulation
  xi_e   += (e - e_ref) * dt
  xi_psi += (Ψ - Ψ_ref) * dt
        │
        ▼
  Augmented state: x_aug = [e, θ, Ψ, ė, θ̇, Ψ̇, xi_e, xi_psi]
        │
        ▼
  Matrix Multiply: tau = -K × x_aug
  (NI A x Vector.vi, K loaded from export_data.csv at startup)
        │
        ▼
  Force decomposition:
  F1 = (u + u0 + v) / 2
  F2 = (u + u0 - v) / 2
        │
        ▼
  Map to ESC: PWM_i = map(Fi, F_min, F_max, ESC_MIN, ESC_MAX)
  Clamp to [1000, 1700] µs
        │
        ▼
  Serial TX: "<esc1>,<esc2>\n"
```

## Loading the Gain Matrix

The VI reads `export_data.csv` (generated by `LQRI.m`) at startup using the
**Read From Spreadsheet File** function. The file must be placed in the same
directory as the VI, or the path control on the front panel must be updated.

The matrix is loaded as a 2×8 numeric 2D array and wired to the K terminal
of the matrix multiply subVI.

## Serial Port Configuration

The VISA Configure Serial Port is set to:
- Baud rate: 230400
- Data bits: 8
- Stop bits: 1
- Parity: None
- Flow control: None

## Running the VI

1. Ensure the Arduino is connected and the correct COM port is selected
2. Verify `export_data.csv` is present and up to date
3. Power the motor supply rail
4. Run the VI — the Arduino ESC calibration sequence will complete first
5. Enable the control loop via the front panel toggle
6. Adjust setpoints as needed

## Known Limitations

- The VI does not implement gain scheduling. The single K matrix applies
  across the full operating range of the linearized model.
- Large step changes in setpoint may cause transient ESC saturation.
  It is recommended to ramp setpoints gradually.
- The integral states are reset to zero when the control loop is disabled.
